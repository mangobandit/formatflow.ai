# Security Policy

Thank you for helping keep FormatFlow safe. We appreciate responsible disclosure of security vulnerabilities.

## Supported Versions

FormatFlow is maintained from the `main` branch and the latest public release.

| Version / branch | Supported |
| --- | --- |
| `main` | ✅ |
| Latest release | ✅ |
| Older releases, archived branches, or forks | ❌ unless explicitly stated in release notes |

## Reporting a Vulnerability

Please **do not open a public GitHub issue** for security vulnerabilities.

Preferred method: use GitHub's **Private vulnerability reporting** / **Report a vulnerability** option for this repository.

If private vulnerability reporting is unavailable, contact: `security@formatflow.ai`.

> Before publishing this policy, make sure `security@formatflow.ai` exists and is monitored. If it does not exist yet, remove the email line and use GitHub private vulnerability reporting only.

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce it
- A proof of concept, screenshots, or logs if helpful
- The affected branch, commit, package, or version
- The possible impact
- Any suggested fix or mitigation, if known
- Your preferred contact information

Please do not include real user data, production secrets, API keys, passwords, or private tokens in your report.

## Response Timeline

We aim to:

- Acknowledge valid reports within **3 business days**
- Triage the issue within **7 business days**
- Provide status updates as we investigate and fix the issue
- Coordinate disclosure once a fix or mitigation is available

Fix timelines depend on severity, complexity, and whether a safe workaround is available.

## Scope

In scope:

- Source code in this repository
- Security issues affecting FormatFlow-owned code, workflows, or deployment configuration
- Vulnerabilities that could affect users, data, authentication, authorization, secrets, or availability

Out of scope:

- Social engineering
- Physical attacks
- Spam or phishing
- Denial-of-service testing
- Issues in third-party services not controlled by FormatFlow
- Reports from automated scanners without a clear, exploitable impact

## Safe Harbor

Good-faith security research that follows this policy will be treated as authorized. Please avoid privacy violations, data destruction, service disruption, or accessing data that does not belong to you.

If you accidentally access sensitive information, stop testing immediately and report the issue.

## Disclosure

Please give us a reasonable opportunity to investigate and fix the issue before publicly disclosing it. We are happy to credit researchers for valid reports if they want recognition.
